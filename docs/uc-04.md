# UC-04: Code Review (Full-Repo PR Review)

## What this does

Review a change against the whole-repo model, not just the diff hunk.

## What you gain

APPROVE/BLOCK verdict with blast radius, missing tests, and evidence.

## How to test (2 steps)

**Step 1 — Terminal** (see the real behavior):

```bash
python scripts/run-lab.py --uc 04
```

Runs compiler order-by/group-by logic that the PR touches.

**Step 2 — Cursor** (paste the prompt below into chat after RubberDuck MCP is connected):

- Index once: `Index my local project at: <path-to-this-repo>`
- **Edit only:** Point `Review diff` at `fixtures/uc-04-pr-order-by-diff.md` or paste the diff.
- Copy the entire **Playground prompt** block below

## What a good answer looks like

Verdict with file:line; diff treated as diff evidence, not applied source.

Must include: repo name, branch/ref, **pinned commit SHA**, and **file:line** evidence.

**Focus files in this repo:** `fixtures/uc-04-pr-order-by-diff.md` + `demoapp/db/query.py`

Review only; do not edit the repo. Required artifact: `CODE_REVIEW.md`.
