# UC-05: Change Impact Analysis (Change Impact)

## What this does

Before you rename or change shared code, see every caller and test you must touch.

## What you gain

Impact map, safe change order, risk score, and out-of-scope residual.

## How to test (2 steps)

**Step 1 — Terminal** (see the real behavior):

```bash
python scripts/run-lab.py --uc 05
```

Shows `config_values` and mirrored `values` dict.

**Step 2 — Cursor** (paste the prompt below into chat after RubberDuck MCP is connected):

- Index once: `Index my local project at: <path-to-this-repo>`
- **Edit only:** Fill `Proposed change` (example: rename `config_values` → `values`).
- Copy the entire **Playground prompt** block below

## What a good answer looks like

`IMPACT_REPORT.md` listing consumers of `demoapp/config.py`.

Must include: repo name, branch/ref, **pinned commit SHA**, and **file:line** evidence.

**Focus files in this repo:** `demoapp/config.py` — rename `config_values`

---

Repository `RubberDuck-com/rubberduck-use-case-playground`;

Read-only impact analysis; required artifact `IMPACT_REPORT.md`.
Use prod RD current-ref gates: `get_started`, full `detailed_repo_analysis(..., semantic_mode="full")`, Phase 2 ready, `load_code(max_files=4000)`.
Stop if change input, ref binding, or hard RD gates fail; reject stale/wrong-commit evidence and web/raw GitHub fallback.

Validate API/state/data impacts, compatibility risk, and safe sequencing with source plus targeted graph/trace/search/security evidence.
