# UC-08: Check Code Logic (Business Logic Check)

## What this does

Does a stated rule actually hold in the code?

## What you gain

Yes/no verdict with the enforcing (or missing) guard at file:line.

## How to test (2 steps)

**Step 1 — Terminal** (see the real behavior):

```bash
python scripts/run-lab.py --uc 08
```

Runs staleness logic live.

**Step 2 — Cursor** (paste the prompt below into chat after RubberDuck MCP is connected):

- Index once: `Index my local project at: <path-to-this-repo>`
- **Edit only:** Fill `Logic question` about `get_outdated_docs` freshness behavior.
- Copy the entire **Playground prompt** block below

## What a good answer looks like

`LOGIC_CHECK.md` with branch map and verdict.

Must include: repo name, branch/ref, **pinned commit SHA**, and **file:line** evidence.

**Focus files in this repo:** `demoapp/builders/html.py` — `get_outdated_docs`
