# UC-03: Localize and Fix Bugs (Bug Localization)

## What this does

Trace a symptom or failing test to root cause in the code graph.

## What you gain

Root cause, evidence table, blast radius, minimal fix direction — each step cited at file:line.

## How to test (2 steps)

**Step 1 — Terminal** (see the real behavior):

```bash
python scripts/run-lab.py --uc 03
```

Shows the live ORM bug: wrong aggregation count and mutated mask.

**Step 2 — Cursor** (paste the prompt below into chat after RubberDuck MCP is connected):

- Index once: `Index my local project at: <path-to-this-repo>`
- **Edit only:** Fill `Bug signal` with the symptom (example provided below).
- Copy the entire **Playground prompt** block below

## What a good answer looks like

`BUG_LOCALIZATION_REPORT.md` pointing at `demoapp/db/query.py` aggregation logic.

Must include: repo name, branch/ref, **pinned commit SHA**, and **file:line** evidence.

**Focus files in this repo:** `demoapp/db/query.py` — `get_aggregation`, `rewrite_cols`

---
