# UC-01: Understand Your Code (Codebase Atlas)

## What this does

Map an unfamiliar repo: entry points, call chains, data flows, architecture — pinned to a commit.

## What you gain

You get an onboarding report so you (or an agent) know where the app starts and how code connects — with file:line proof, not guesses.

## How to test (2 steps)

**Step 1 — Terminal** (see the real behavior):

```bash
python scripts/run-lab.py --uc 01
```

Builds a real HTML file at `sampledocs/_build/index.html`.

**Step 2 — Cursor** (paste the prompt below into chat after RubberDuck MCP is connected):

- Index once: `Index my local project at: <path-to-this-repo>`
- **Edit only:** Usually nothing for this playground — the Target below is pre-filled.
- Copy the entire **Playground prompt** block below

## What a good answer looks like

Report names repo + pinned SHA, lists entry points (e.g. `demoapp/cmd/build.py:main`), call chains, and data flows.

Must include: repo name, branch/ref, **pinned commit SHA**, and **file:line** evidence.

**Focus files in this repo:** `demoapp/cmd/build.py` (main), `demoapp/application.py` (Application)
